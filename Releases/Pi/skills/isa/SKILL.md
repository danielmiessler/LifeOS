# ISA — Ideal State Artifact

## Purpose
Define the ideal end state of a system/project. Not what exists today — what should exist.

## 12-Section Template

1. **Executive Summary** — One-paragraph vision statement
2. **Context & Drivers** — Why this ideal state matters
3. **Scope & Boundaries** — What is in/out of scope
4. **Principles & Constraints** — Governing rules
5. **Capability Model** — What the system must do
6. **Structural Architecture** — How it is built
7. **Data Model** — Key entities and relationships
8. **User Experience** — How users interact
9. **Integration Points** — External connections
10. **Quality Attributes** — Performance, security, reliability
11. **Evolution Path** — How to get there incrementally
12. **Success Criteria** — Measurable outcomes

## ISC Rules

- **Specific**: No vague statements. Each claim is testable.
- **Coherent**: All sections reinforce each other.
- **Minimal**: Omit nothing essential, include nothing extraneous.
- **Temporal**: Distinguish now vs. later vs. never.

## Scaffolding Guidance

Start with section 2 (Context) and 5 (Capabilities). Let those drive the rest. Fill sections 1 and 12 last — they summarize everything above.
