# Installing DriftMon

Welcome to DriftMon. This guide walks you through installing the AI behavioral drift detector and its optional SessionEnd hook.

## Phase 1: System Analysis

Run these checks to confirm your environment is ready.

```bash
# Check Python 3 is installed
python3 --version

# Check that required Python modules are available (all stdlib)
python3 -c "import re, os, sys, json, math, statistics, csv; print('All modules available')"

# Check that bun is installed (needed for the SessionEnd hook)
bun --version

# Check for existing learning files (DriftMon's input)
find ~/.claude/MEMORY/LEARNING -name "*.md" 2>/dev/null | wc -l
# Note: DriftMon needs learning files to build a baseline.
# If this returns 0, you need to accumulate sessions before DriftMon is useful.
```

**Requirements:**
- Python 3 (any recent version, uses only stdlib modules)
- bun runtime (for the SessionEnd hook only)
- Learning files in `~/.claude/MEMORY/LEARNING/` (generated by WorkCompletionLearning or manually)

## Phase 2: User Questions

Before installing, decide:

1. **Automatic or manual?** The SessionEnd hook runs DriftMon automatically after every session. If you prefer to run it manually, skip the hook installation.

2. **CSV output path:** By default, results go to `/root/driftmon_results.csv`. Edit the `OUTPUT_CSV` variable in `driftmon.py` if you want a different location.

3. **ChatGPT analysis?** If you have ChatGPT exports in searchable markdown format, DriftMon can analyze those too with `--mirror`. No extra setup needed.

## Phase 3: Installation

### Core Analyzer

```bash
# Create destination directory
mkdir -p ~/.claude/hooks

# Copy the analyzer
cp src/Tools/driftmon.py ~/.claude/hooks/driftmon.py

# Make it executable
chmod +x ~/.claude/hooks/driftmon.py

# Verify it runs
python3 ~/.claude/hooks/driftmon.py --help 2>&1 || echo "Check Python installation"
```

### SessionEnd Hook (Optional)

If you want DriftMon to run automatically at session end:

```bash
# Copy the hook
cp src/Tools/DriftMonRun.hook.ts ~/.claude/hooks/DriftMonRun.hook.ts

# Make it executable
chmod +x ~/.claude/hooks/DriftMonRun.hook.ts
```

Update the Python path inside `DriftMonRun.hook.ts` to match where you installed `driftmon.py`:

```typescript
// Change this line to match your installation path:
const proc = spawn("python3", ["~/.claude/hooks/driftmon.py", "--csv"], {
```

Add the hook to your Claude Code hooks configuration in `.claude/settings.json`:

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "command": "bun run ~/.claude/hooks/DriftMonRun.hook.ts",
        "timeout": 30000
      }
    ]
  }
}
```

If you also use WorkCompletionLearning, place DriftMonRun after it in the SessionEnd array so new learnings are captured before analysis runs.

### Input Directory Configuration

DriftMon auto-discovers learning directories under `~/.claude/MEMORY/LEARNING/`. If your files are elsewhere, edit the `_discover_learning_dirs()` function in `driftmon.py`:

```python
def _discover_learning_dirs():
    base = Path("~/.claude/MEMORY/LEARNING").expanduser()  # Change this path
    # ...
```
