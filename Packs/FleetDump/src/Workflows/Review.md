# Workflow: Review

Read and interpret an existing fleet dump to assess patterns, pace, and sustainability.

## Steps

### 1. Locate the dump file

Determine which date to review:

- Today: look for `/mnt/g/My Drive/fleet-dumps/YYYY-MM-DD.md` with today's date
- Yesterday: use yesterday's date
- Specific date: use the date the user provides

If the file doesn't exist, offer to run the Dump workflow first.

### 2. Read the dump

Read the full markdown file and parse each section.

### 3. Assess session patterns

**Duration:**
- Under 4 hours: light day, probably fine
- 4-8 hours: normal working session
- 8-12 hours: heavy day, check if this is a pattern
- Over 12 hours: flag as potential regulation drift

**Commit volume:**
- 1-5 commits: normal day
- 6-10 commits: active day
- 11-20 commits: high-output day, check sustainability
- Over 20: flag as unsustainable unless it's a major launch

**Code churn:**
- Over 1000 lines added in a day: high volume, verify quality
- Over 500 lines deleted: significant refactor or cleanup, verify intentional
- High additions AND high deletions: major rework, check if this is thrashing

### 4. Check governor flags

Review any pace flags generated by the tool:

- HIGH COMMIT VOLUME: more than 10 commits in one day
- LARGE CODE VOLUME: over 1000 lines added
- LARGE DELETION: over 500 lines removed

These are signals, not judgments. Context matters. A migration day legitimately produces high churn.

### 5. Review flinch log

If the flinch log has unresolved entries:

- 1-2 entries: normal, address when convenient
- 3+ entries: pattern forming, should be discussed
- Same category recurring: systemic issue, escalate

### 6. Cross-reference with vitals (if available)

Check if a vitals dump exists for the same date at `/mnt/g/My Drive/fleet-governor/vitals/YYYY-MM-DD.md`.

If it exists, read it and correlate:

| Fleet Signal | Vitals Signal | Interpretation |
|---|---|---|
| High commits | Low sleep | Running on fumes -- unsustainable |
| Long session (8h+) | Low Body Battery | Body is paying for the mind's ambition |
| Weekend activity | Low HRV | Regulation drift -- escalate |
| Normal pace | High readiness | Green light -- capacity available |
| Light day | Low readiness | Recovery working as intended |

### 7. Present the review

Summarize with:

1. **One-line verdict:** "Sustainable day" or "Heavy but contained" or "Pace flag -- check in"
2. **Key numbers:** Session span, commit count, line changes
3. **Flags:** Any pace or flinch concerns
4. **Vitals correlation:** If available, the body-behavior connection
5. **Governor questions:** Read out the four questions and offer initial answers based on the data

## When to Escalate

Flag to the user if any of these are true:

- Session span exceeds 12 hours
- Commit volume exceeds 20 in a single day
- Flinch log has 3+ unresolved entries
- Two consecutive days with pace flags
- Fleet dump + vitals dump both show warning signals
